# Chinese Sentence Flashcard App — Design Spec

## Context

The user is a heritage Chinese speaker (grew up in Canada) who is comfortable speaking Chinese at home but never learned to read/write. They are learning characters daily with their parents and need a tool to practice reading sentences. Commercial apps don't fit because: (1) they need sentence-level practice with per-character pinyin support, (2) they want to use LLM-generated content tailored to their known vocabulary, and (3) they want to avoid API costs.

## Overview

A static (no server) Anki-like flashcard app for practicing Chinese sentence reading. Flashcard sets are JSON files generated by pasting LLM output into a built-in parser. Characters the user hasn't learned yet display pinyin annotations above them. Uses a simplified SRS algorithm for session-based review.

## Tech Stack

- Plain HTML / CSS / JavaScript — no frameworks, no build tools
- Served via a minimal local server (e.g., `python -m http.server 8000` from the project root) — required because browsers block `fetch()` on `file://` URLs. Just run the command and open `http://localhost:8000`.
- A small Python script to regenerate the set index when new sets are added

## Data Formats

### Learned Characters (`data/learned_characters.txt`)

A flat string of Chinese characters, no separators. Each character is one Unicode codepoint.

```
我你他她好吃喝看书学
```

### LLM Output (what the user pastes)

```json
{
  "sentences": ["我今天去超市买东西"],
  "pinyin": ["wǒ jīn tiān qù chāo shì mǎi dōng xi"]
}
```

- `sentences`: array of Chinese sentences (no spaces)
- `pinyin`: array of space-separated pinyin strings with accented vowels, one entry per sentence, one syllable per character (punctuation excluded)

### Parsed Flashcard Set (`data/sets/<name>.json`)

```json
{
  "name": "grocery-shopping",
  "created": "2026-03-31",
  "sentences": ["我今天去超市买东西"],
  "pinyin": ["wǒ jīn tiān qù chāo shì mǎi dōng xi"],
  "cards": [
    {
      "sentence": "我今天去超市买东西",
      "characters": [
        {"char": "我", "pinyin": "wǒ", "learned": true},
        {"char": "今", "pinyin": "jīn", "learned": false},
        {"char": "天", "pinyin": "tiān", "learned": false},
        {"char": "去", "pinyin": "qù", "learned": true},
        {"char": "超", "pinyin": "chāo", "learned": false},
        {"char": "市", "pinyin": "shì", "learned": false},
        {"char": "买", "pinyin": "mǎi", "learned": true},
        {"char": "东", "pinyin": "dōng", "learned": false},
        {"char": "西", "pinyin": "xi", "learned": false}
      ]
    }
  ]
}
```

- `sentences` and `pinyin` are preserved at the top level so reparse can rebuild `cards` from scratch
- Non-Chinese characters (punctuation, spaces) are excluded from the `characters` array
- The parser matches pinyin syllables to Chinese characters only, skipping punctuation

### Set Index (`data/index.json`)

```json
{
  "sets": [
    {"name": "grocery-shopping", "file": "grocery-shopping.json", "created": "2026-03-31"},
    {"name": "family-dinner", "file": "family-dinner.json", "created": "2026-03-30"}
  ]
}
```

Generated by `scripts/build_index.py`.

## UI Views

All three views live in `index.html`, toggled via JavaScript.

### 1. Set Selection (Home)

- Lists all flashcard sets from `index.json` with name and creation date
- Click a set to start studying
- Two action buttons at the top: **"Add New Set"** and **"Reparse All Sets"**

### 2. Study View

**Front of card (initial state):**
- Sentence displayed with large characters
- Unlearned characters have pinyin shown above them using ruby text (`<ruby>` / `<rt>` HTML elements)
- Learned characters show no annotation
- All characters are clickable at all times (before and after revealing answer)
- "Show Answer" button below the card

**Back of card (after "Show Answer"):**
- Full pinyin revealed above every character (both learned and unlearned)
- Three SRS buttons: **Again** / **Good** / **Easy**

**Character click behavior:**
- Clicking a character highlights it (e.g., red underline or background color change)
- All cards in the current set containing that character are collected, deduplicated, and appended to the end of the queue (if not already in the queue)
- A small toast notification appears: "Requeued N cards with 超"

**Progress & navigation:**
- Progress counter showing position in queue (e.g., "3 / 12")
- Back button to return to set selection
- Completion screen when queue is empty — shows summary stats (cards studied, characters flagged) and a button to go back

### 3. Add Set View

- Text area to paste LLM JSON output
- Input field for set name
- **"Parse & Download"** button:
  - Parses the pasted JSON
  - Fetches `data/learned_characters.txt` from the server automatically
  - Maps each Chinese character to its pinyin syllable, sets `learned` flag by cross-referencing learned characters
  - Downloads the resulting flashcard set JSON file
  - User manually moves the file to `data/sets/` and runs `build_index.py`

- **"Reparse All Sets"** button:
  - Fetches `data/learned_characters.txt` and all sets listed in `data/index.json` from the server
  - Reparses each set: rebuilds `cards` array from the preserved `sentences` and `pinyin` using the updated learned characters
  - Downloads a zip or individual updated set JSON files

## SRS Algorithm (Session-Scoped)

Simplified, session-only algorithm (no cross-day scheduling).

Each card tracks:
- `interval` (starts at 1)
- `streak` (starts at 0)

### Button behavior:

- **Again** — reset `interval` to 1, reset `streak` to 0. Card is inserted a few positions ahead in the queue (not immediately next, to allow spacing).
- **Good** — `interval` × 2, `streak` + 1. Card is inserted `interval` positions ahead in the queue. If that exceeds the queue length, card is done.
- **Easy** — card is removed from the queue entirely.

### Character requeue:

When a character is clicked:
1. Find all cards in the set containing that character
2. Filter to cards not already in the queue
3. Append them to the end of the queue
4. Show toast: "Requeued N cards with [char]"

## Parser Logic

The parser's job is to align Chinese characters with pinyin syllables:

1. Iterate through each sentence and its corresponding pinyin string
2. Split pinyin by spaces into syllables
3. Iterate through the sentence character by character:
   - If the character is a Chinese character (Unicode range check): consume the next pinyin syllable, look up `learned` status from the learned characters set
   - If the character is punctuation/space/number: skip it (do not consume a pinyin syllable)
4. Produce the `characters` array for that card

## `build_index.py` Script

- Scans `data/sets/*.json`
- Reads `name` and `created` from each file
- Writes `data/index.json` with the list of sets
- Run manually after adding/updating sets: `python scripts/build_index.py`

## Verification Plan

1. Create a sample `learned_characters.txt` with a few characters
2. Create a test LLM output JSON and use the Add Set view to parse it
3. Verify the downloaded JSON has correct per-character pinyin and learned flags
4. Move the file to `data/sets/`, run `build_index.py`, refresh the page
5. Verify the set appears in the home view
6. Study the set: confirm pinyin shows only on unlearned characters, "Show Answer" reveals all pinyin
7. Test SRS buttons: Again requeues near front, Good spaces out, Easy removes
8. Test character click: confirm toast appears and cards are requeued
9. Test reparse: add a character to learned list, reparse, verify it updates
10. Test completion screen appears when queue is empty
